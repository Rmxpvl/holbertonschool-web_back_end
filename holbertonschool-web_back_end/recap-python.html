Python Fundamentals

Conditions & boucles

Piloter l'exécution d'un programme.

Data structures

Structures de données

  • Regrouper des fonctions liées dans un module
  • Réimplémentation d'algorithmes simples (pow)
Exceptions

Gestion des erreurs

Rendre les scripts robustes face aux entrées invalides.

  • try / except / finally
  • raise avec message personnalisé
  • Exceptions courantes : TypeError, ValueError, IndexError, ZeroDivisionError, NameError
  • Retour de statut (True/False, compteurs) plutôt qu'un crash

Serialization

Sérialisation de données

Convertir des objets Python ↔ formats stockables.

  • JSON : json.dump/load, dumps/loads
  • Pickle : objets Python complets, "wb"/"rb", @classmethod
  • CSV : csv.DictReader, conversion vers JSON
  • XML : xml.etree.ElementTree, arbre de balises
  • ⚠️ Sécurité : ne jamais pickle.load une source non fiable
ORM

Object-Relational Mapping

Du SQL brut (MySQLdb) à SQLAlchemy.

  • Connexion & curseur avec MySQLdb
  • Requêtes paramétrées (%s) contre les injections SQL
  • JOIN entre tables
  • Modèles SQLAlchemy : Column, ForeignKey, declarative_base
  • session.query/filter/order_by, CRUD complet + commit()
  • Relations entre modèles via .join()
REST API

API REST

De http.server à Flask, avec authentification.

  • Rappels HTTP : méthodes, codes de statut, headers
  • Client API avec requests
  • Serveur HTTP "à la main" (BaseHTTPRequestHandler)

  • Principes REST : ressources, verbes HTTP, stateless

Conception logicielle

Design patterns

Factory · Observer · Decorator

  • Factory : centraliser la création d'objets via un registre
  • Observer : un sujet notifie plusieurs observateurs (découplage émetteur/récepteurs)
  • Decorator : composer des comportements sans multiplier les sous-classes
